# 伴奏生成调研

## 目标

伴奏生成要为练习提供稳定、可循环、可变速、可编辑、可同步的背景声部。第一阶段建议做规则和 pattern 系统，不直接做 AI 音频生成。

## 基本原理

伴奏生成的核心不是“直接生成一段音频”，而是把音乐结构逐步展开成可播放事件：

```text
和弦进行
  -> 曲式段落
  -> 风格 Pattern
  -> 乐器声部
  -> MIDI 事件
  -> 音源播放
```

输入通常包括：

- 调性。
- 拍号。
- 速度。
- 和弦进行。
- 曲式段落。
- 风格。
- 密度和难度。

输出通常是多声部事件：

- 鼓。
- 贝斯。
- 钢琴。
- 吉他扫弦。
- 吉他分解。
- Pad。
- 简单填充旋律。

## Pattern 是什么

Pattern 是“某种风格在一小节或数小节内怎样运动”的模板。它不直接绑定绝对音高，而是绑定节拍位置、力度、声部角色和相对音。

例子：

```text
4/4 Pop Bass Pattern
beat 1: root
beat 2: fifth
beat 3: root
beat 4: fifth or approach note
```

当和弦是 `C` 时，root 是 C，fifth 是 G。当和弦变成 `Am` 时，root 是 A，fifth 是 E。这样同一个 pattern 可以复用到不同和弦。

## 各声部生成逻辑

### 鼓

鼓通常不依赖和弦，主要依赖风格、拍号和段落强度。

- Kick 强调重拍。
- Snare/Clap 强调 backbeat。
- Hi-hat 或 ride 决定细分节奏。
- Fill-in 用于段落结尾和过门。

### 贝斯

贝斯依赖和弦根音、五音、经过音和节奏风格。

- 简单模式：root / fifth。
- 流行模式：根音 + 八分音符律动。
- Blues/Jazz：walking bass 和半音趋近。

### 钢琴

钢琴常用于和声铺底或节奏型伴奏。

- Block chord。
- 分解和弦。
- 左手低音 + 右手和弦。
- syncopation。

### 吉他

吉他伴奏要区分扫弦和分解。

- 扫弦：上下方向、闷音、重音、空拍。
- 分解：弦序、和弦指型、可演奏性。

## 为什么先不用 AI 音频生成

- 可控性差。
- 很难和谱面小节精确同步。
- 难以编辑某个声部或某个小节。
- 变速和循环练习不稳定。
- 生成结果难以复现。

练习软件更需要可解释的音乐结构，而不是一次性音频结果。

## 数据结构重点

伴奏模型至少需要：

- chord progression。
- section。
- style。
- pattern id。
- bar range。
- instrument part。
- rhythmic grid。
- voicing rule。
- velocity rule。
- humanization。
- fill rule。

## 第一阶段建议

- 从 4/4、Pop、Folk、Blues 三类风格开始。
- 先做鼓、贝斯、钢琴三个声部。
- 输出 MIDI 事件，不直接输出音频文件。
- 支持小节循环、变速、静音声部。
- 让伴奏和谱面共享同一套小节时间轴。

## 第一阶段 MVP 工作拆解

第一阶段不要先做完整自动编曲系统，而是做一个可验证闭环：

```text
用户输入和弦进行 -> 选择风格 -> 生成多声部 MIDI 事件 -> 播放/循环/变速 -> 与谱面小节同步
```

需要完成的工作：

- 和弦进行输入：支持按小节输入 `C`、`Am`、`F`、`G7` 这类基础和弦，并能保存小节范围。
- 和弦解析：把和弦字符串解析成 root、quality、extension、bass 等结构化字段。
- 时间轴模型：建立 bar、beat、tick 的统一时间表示，伴奏和谱面共用。
- Pattern 数据格式：定义鼓、贝斯、钢琴 pattern 的节拍位置、力度、相对音和重复规则。
- Pattern 展开器：把 pattern 根据当前和弦展开成具体 MIDI note event。
- 基础风格库：先实现 2 到 3 个 4/4 风格，例如 Pop、Folk、Blues，每个风格覆盖鼓、贝斯、钢琴。
- MIDI 播放链路：把生成结果送到浏览器或桌面端音源，支持开始、暂停、停止。
- 练习控制：支持小节循环、变速、声部静音、简单 metronome。
- 导出与调试：能导出或打印生成的 MIDI 事件，方便检查伴奏是否落在正确拍点。

其中几个容易混淆的概念需要先定义清楚：

- 和弦解析：不是只把 `Cmaj7/G` 当作一段文本保存，而是拆成可计算字段。`root` 是根音，例如 `C`；`quality` 是和弦性质，例如 major、minor、dominant；`extension` 是七音、九音、十一音、十三音等扩展音；`bass` 是 slash chord 里的低音，例如 `C/G` 的 bass 是 `G`。拆开以后，系统才能正确移调、生成贝斯音、生成钢琴和弦音，并在谱面上规范化显示。
- 练习控制：指练习时对伴奏播放方式的控制，而不是改变和弦本身。小节循环用于反复练难点小节；变速用于慢练或逐步提速；声部静音可以只听鼓、只听贝斯，或关掉钢琴；metronome 是节拍器，用于在伴奏很简单或关闭部分声部时仍然保持稳定拍点。

暂时不进入 MVP：

- AI 音频生成。
- 自动生成复杂前奏、尾奏和过门。
- 复杂爵士 walking bass。
- 吉他真实扫弦拟真。
- 多风格大型风格库。

这些能力后置的原因是它们会明显扩大第一阶段的音乐规则和产品复杂度。前奏、尾奏和过门需要理解段落结构和张力变化，不能只按每小节和弦机械展开；walking bass 需要处理和弦连接、半音趋近和声部进行，错误时会比简单 root/fifth 贝斯更明显；吉他真实扫弦涉及上下扫方向、闷音、弦序、指型和可演奏性；大型风格库则需要大量 pattern 数据、试听校验和风格命名规则。第一阶段先把“和弦表 + 少量风格 + MIDI 事件 + 同步播放”的闭环做可靠，再扩展这些更像编曲系统的能力。

## 可参考产品

- iReal Pro：和弦表驱动伴奏。
- Band-in-a-Box：老牌自动伴奏软件。用户输入和弦进行、曲式和风格后，软件根据风格库生成鼓、贝斯、钢琴、吉他等声部。这里主要借鉴“和弦表 + 风格库 -> 多声部伴奏”的产品思路，不需要第一阶段复刻它的大型风格库和复杂编曲能力。
- Guitar Pro RSE：Guitar Pro 的 Realistic Sound Engine，用于让谱面播放听起来更接近真实乐器，而不是普通 General MIDI 音色。这里主要借鉴“谱面事件驱动播放”和“不同乐器需要不同演奏效果”的思路，不把 RSE 当作伴奏生成模型。
- DAW MIDI pattern / arranger track：DAW 是 Digital Audio Workstation，例如 Logic Pro、Ableton Live、Cubase。MIDI pattern 是可重复的 MIDI 片段，arranger track 用来按 Intro、Verse、Chorus 等段落组织音乐。这里主要借鉴“片段复用”和“段落编排”的工作流。
