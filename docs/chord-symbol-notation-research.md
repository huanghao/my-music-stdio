# 和弦表示法调研

## 目标

这篇文档回答两个问题：

- 和弦有没有比较通用、标准的表示方法。
- 做音乐练习工作站前，是否需要先学习这些表示法。

结论：需要先学基础，但不需要一开始覆盖所有理论体系。第一阶段建议支持流行/爵士和弦符号，例如 `C`、`Cm`、`C7`、`Cmaj7`、`Cm7b5`、`C/G`，并在内部模型中拆成结构化字段。罗马数字分析和 Nashville Number System 更适合做和声分析、移调和教学解释，不适合直接替代和弦符号输入。

## 常见表示体系

### 流行/爵士和弦符号

用途：

- 歌谱、和弦谱、Real Book、吉他谱、钢琴伴奏谱。
- 用户最常见的输入和显示形式。

例子：

- `C`：C 大三和弦。
- `Cm`：C 小三和弦。
- `C7`：C 属七和弦。
- `Cmaj7`：C 大七和弦。
- `Cm7b5`：C 半减七和弦。
- `Cadd9`：C 加九音。
- `C/G`：C 和弦，G 在低音。

优点：

- 实用、普及、适合用户输入。
- 与 ChordPro、iReal Pro 类产品、吉他谱生态容易对接。

风险：

- 不是一个完全统一的国际标准。
- 同一个和弦可能有多种写法，例如 `Cmaj7`、`CM7`、`CΔ7`。
- 扩展音、省略音、变化音、转位、slash bass 的解析需要明确规则。

### iReal Pro 的和弦表示法

iReal Pro 是一个常见的和弦谱和自动伴奏应用。它的谱面主要记录和弦进行、段落、反复和风格伴奏信息，不记录完整旋律。因此它的和弦表示法很适合作为“和弦表驱动伴奏”的参考。

iReal Pro 使用的是偏爵士/流行语境的和弦符号体系，但它不是通用标准本身，而是应用内部支持的一组写法。它的特点是：

- 根音 + 和弦性质，例如 `C`、`C-`、`C7`、`C^7`、`C-7`。
- 支持扩展和变化音，例如 `C7b9`、`C7#11`、`C-7b5`。
- 支持 slash chord，例如 `C/E`，表示 C 和弦但 E 在低音。
- 支持 `N.C.`，表示 no chord，不播放和弦伴奏。
- 更关注和弦进行和伴奏播放，不表达旋律、指法或完整谱面排版。

对本项目的启发：

- 可以学习它把和弦表、曲式和伴奏风格绑定在一起的方式。
- 不应直接依赖 iReal Pro 的私有格式作为内部模型。
- 可以把 iReal Pro 支持的常见和弦质量当作 MVP 的输入兼容范围之一。
- 用户输入层可以接受 `Cmaj7`、`C^7`、`C-7` 这类等价写法，再规范化成内部结构。

### 罗马数字分析

用途：

- 和声功能分析。
- 调内级数和离调分析。
- 教学解释和移调。

例子：

- C 大调中的 `C - F - G7 - C` 可分析为 `I - IV - V7 - I`。
- A 小调中的 `E7` 可分析为 `V7` 或和声小调语境下的属和弦。

优点：

- 能表达“这个和弦在调性中的功能”。
- 适合移调，因为 `I - IV - V` 可以映射到任意调。

风险：

- 依赖调性上下文。
- 古典、爵士、流行音乐中的标记习惯不完全一致。
- 不适合直接作为用户最初输入格式。

### Nashville Number System

用途：

- 乐队排练、现场演出、快速移调。
- 用数字表达调内级数和节奏布局。

Nashville 指美国田纳西州的 Nashville（纳什维尔），它是美国乡村音乐和录音工业的重要中心。Nashville Number System 可以翻译成“纳什维尔数字记谱法”。它不是某个软件名，而是一套面向乐手排练和现场演出的和弦数字标记方法。

例子：

- C 大调的 `C - Am - F - G` 可写为 `1 - 6m - 4 - 5`。
- 换到 D 大调后变为 `D - Bm - G - A`，数字不变。

优点：

- 非常适合伴奏生成和移调。
- 比罗马数字更贴近流行音乐排练语境。

风险：

- 对初学用户不如和弦名直观。
- 复杂转调、借用和弦、扩展和弦仍需要附加规则。

## 内部模型建议

不要只保存和弦字符串。建议至少拆成：

```text
ChordSymbol
  root: C
  quality: major | minor | dominant | diminished | augmented | suspended | power
  extensions: 7 | maj7 | 9 | 11 | 13
  alterations: b5 | #5 | b9 | #9 | #11 | b13
  additions: add9 | add11
  omissions: no3 | no5
  bass: optional slash bass
  originalText: 原始输入
```

字段含义：

- `root`：和弦根音，也就是和弦命名的基础音。例如 `Cmaj7` 的 root 是 `C`，`F#m7b5` 的 root 是 `F#`。
- `quality`：和弦性质，描述根音上方最核心的三和弦或基础结构。`major` 表示大三和弦，`minor` 表示小三和弦，`dominant` 通常表示带小七音的属和弦，`diminished` 表示减和弦，`augmented` 表示增和弦，`suspended` 表示 sus 类和弦，`power` 表示只有根音和五音的强力和弦。
- `extensions`：在基础三和弦或七和弦之外继续叠加的音，例如 `7`、`maj7`、`9`、`11`、`13`。`C9` 可以理解为 C 属七和弦再加九音。
- `alterations`：变化音，表示把某些标准和弦音升高或降低。例如 `C7b9` 里的 `b9` 表示降九音，`C7#11` 里的 `#11` 表示升十一音，`Cm7b5` 里的 `b5` 表示降五音。
- `additions`：额外添加的音，但不暗示完整扩展和弦结构。例如 `Cadd9` 表示 C 大三和弦加九音，不等同于 `C9`。
- `omissions`：明确省略某个常规和弦音。例如 `C(no3)` 表示省略三音，因此不直接区分大三或小三色彩。
- `bass`：slash chord 的低音字段，表示斜杠后面的低音。例如 `C/E` 的 root 是 `C`，bass 是 `E`，意思是弹 C 和弦但最低音用 E；这通常用于表示转位或指定低音线。
- `originalText`：用户原始输入文本，用来保留用户习惯写法。例如用户输入 `C^7`，内部可以规范化为 `Cmaj7`，但仍保留原始写法用于回显或编辑。

`originalText` 用来保留用户输入风格，结构化字段用于移调、伴奏生成、和声分析和谱面渲染。

## 学习优先级

第一阶段需要掌握：

- 大三和弦、小三和弦、减三和弦、增三和弦。
- 属七、大七、小七、半减七、减七。
- sus2、sus4、add9。
- slash chord，例如 `C/E`、`G/B`。
- 调内级数：`I`、`ii`、`iii`、`IV`、`V`、`vi`、`vii°`。

暂时可以后置：

- 复杂爵士 upper structure。
- 多层 polychord。
- 复杂转调分析。
- 不同教材体系下的等价标记兼容。

## 对产品的影响

- 和弦输入：先支持宽松输入，再规范化显示。
- 移调：对结构化 root 和 bass 做移调，不能简单替换字符串。
- 伴奏生成：根据 root、quality、extensions 选择可用音。
- 练习评分：用户弹到根音、三音、七音、扩展音时，评分权重可以不同。
- 教学解释：同一个 `G7` 可以显示为“G 属七和弦”，也可以在 C 大调中解释为 `V7`。
